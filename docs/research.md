## 1. Топ open-source проектов

## 2. Стратегии, которые реально работают

### Grid Trading -- лучшая для боковика

- **Суть:** сетка лимитных ордеров на покупку и продажу с фиксированным шагом
- **Spacing:** 0.5% для BTC/ETH, 1-2% для альткоинов
- **Geometric grid лучше arithmetic** -- соответствует процентным движениям цены
- **Реальные результаты:** 3-14% в месяц при боковом рынке
- **Пример:** BTC/USDT, 20 гридов, диапазон $35k-$42k, spacing 1% -- 14.2% месячной доходности (Q2 2023)
- **Риск:** при сильном тренде бот набирает одностороннюю позицию
- Не более 20% портфеля на один грид

### Smart DCA (Safety Orders)

- **Суть:** докупка при падении на заданный % от начальной позиции
- **Price deviation (шаг):** например, 1%, 2%, 3%...
- **Volume scale:** множитель объема каждого следующего ордера (1.5x, 2x)
- **Максимум safety orders:** ограничивает риск
- Работает на коррекциях и нисходящих трендах
- **Indicator-based DCA** (вход по RSI < 30, MACD crossover) -- более умные входы

> **Martingale DCA** (удвоение позиции после убытка: 1, 2, 4, 8, 16 лотов) -- **крайне опасная** стратегия. Может уничтожить депозит при затяжном тренде. Только с жестким max-order limit и stop-loss.

### Market Making -- для профессионалов

- **Суть:** постоянное выставление bid/ask ордеров, заработок на спреде
- **Пример:** BTC @ $50,000 -- bid $49,900 / ask $50,100 = $200 на BTC (минус комиссии)
- **Типичные спреды:** 0.1-0.5% на ликвидных парах
- **Капитал:** $5k-$10k минимум, $50k+ для профессионалов
- **Ключевой риск:** inventory risk при направленных движениях
- **Hummingbot** -- лучший open-source инструмент

### Арбитраж

- **Треугольный:** A -> B -> C -> A внутри одной биржи
- **Кросс-биржевой:** разница цен между биржами
- **Реальность 2025:** 240,000+ успешных арбитражных сделок за год, объем ~$868 млн
- **Проблема:** возможности исчезают за миллисекунды. Побеждает самый быстрый. Для розничного трейдера на CEX -- почти невозможно конкурировать с институционалами

### Trailing Stop -- вариации

| Тип | Как работает | Параметры |
|---|---|---|
| **Percentage-based** | Фиксированный % от максимума | 1-3% |
| **ATR-based** | Стоп = ATR * множитель | Множитель 2.5-3.5x |
| **Chandelier Exit** | Highest High - (ATR x Multiplier) | 22 периода ATR, множитель 3.0 |

**Для крипты ATR-based предпочтительнее** -- адаптируется к реальной волатильности, не выбивает на нормальных колебаниях.

### Multi-Timeframe Analysis

- **Принцип:** Higher TF = тренд, Middle TF = сетап, Lower TF = вход
- **Комбинации:**
  - Day trading: 15M / 1H / 4H
  - Swing trading: 1H / 4H / Daily
- Сигналы на 1H, подтверждённые 4H/Daily, имеют значительно более высокий win rate

### ML-стратегии (FreqAI)

- **Классификация/Регрессия:** обучение на 10k+ фич, предсказание направления/величины движения
- **Reinforcement Learning:** агент проходит по историческим свечам (Long/Short/Neutral). Награда через `calculate_reward()`
- **LSTM:** динамическое взвешивание с агрегатным скорингом
- **Важно:** примеры из документации -- для обучения, не для продакшена. Требуют глубокого feature engineering и регулярного переобучения

### Sentiment Analysis

- OctoBot: Google Trends, Reddit интеграция
- Инструменты: FinBERT (open-source NLP), VADER, LunarCrush, Santiment
- Как самостоятельная стратегия -- не доказала устойчивой прибыльности
- Как фильтр/усилитель для других стратегий -- перспективно

## 3. Продвинутые фичи лидеров

### Hyperopt и оптимизация параметров

- **Freqtrade:** Optuna + NSGAIIISampler
- **Лучшие loss-функции:** Sharpe (универсальная), Sortino (для downside risk), Calmar (низкая толерантность к риску)
- **Правила против переобучения:**
  1. Период бэктеста **минимум 6 месяцев**
  2. Оптимизировать **один параметр за раз**
  3. Walk-forward validation: train на одном периоде, test на следующем
  4. MultiMetricHyperOptLoss вместо чистого профита
  5. Paper trading **3-6 месяцев** перед live

### Walk-Forward Testing

- Разделение данных на train/test окна
- Оптимизация на train -> проверка на test -> сдвиг окна -> повтор
- Эмулирует реальные условия: стратегия постоянно сталкивается с "невиданными" данными

## 4. Что реально делает бота прибыльным

### Комиссии -- тихий убийца

- Типичные maker/taker fee: 0.02-0.1%
- При 100 сделках в день с fee 0.1% = ~20% от оборота в месяц на комиссии
- **Maker rebates** критичны для market making -- некоторые биржи платят за лимитные ордера

### Slippage -- разрыв между теорией и практикой

- На ликвидных парах: 0.5-1%
- На альткоинах: 2-5%+
- **Бэктест-результаты завышены на 20-40%** относительно реальной торговли
- Решения: лимитные ордера, торговля на ликвидных парах, разбиение крупных ордеров

### Position Sizing -- главный фактор выживания

- **Правило 2%:** никогда не рисковать более 2% капитала на одну сделку
- **Kelly Criterion:** оптимальный размер = (win_rate * avg_win - (1-win_rate) * avg_loss) / avg_win
- **Half-Kelly на практике:** 75% оптимального роста при 50% меньшем drawdown. Большинство профи используют 1/4 - 1/2 Kelly
- **Daily loss limit:** 2-3% от капитала -- circuit breaker

### Execution Speed

- Арбитражные возможности живут миллисекунды
- Go и Rust боты выигрывают у Python по латентности
- Colocation рядом с серверами биржи даёт преимущество
