# Roadmap: план развития Crypto Trading Bot

> Текущее состояние проекта и план дальнейшей разработки.
> Последнее обновление: 2026-03-26

## Текущее состояние (MVP)

### Что реализовано

**Торговый движок**
- Асинхронный движок с поддержкой Paper Trading и Live Trading
- Подключение к биржам Binance и Bybit через CCXT
- Автоматическое исполнение ордеров (market orders)
- Мониторинг Stop Loss / Take Profit на каждом цикле

**6 торговых стратегий**
- EMA Crossover -- трендовая (EMA 9/21/200)
- RSI Mean Reversion -- контртрендовая (RSI + Bollinger Bands)
- Grid Trading -- сеточная торговля для бокового рынка
- Smart DCA -- умное усреднение с подтверждением индикаторов
- Supertrend -- агрессивная трендовая (Supertrend + ADX)
- Multi Indicator -- консенсус 6 индикаторов (стратегия по умолчанию)

**Risk Management**
- 3 пресета риска: Conservative / Moderate / Aggressive
- Per-trade Stop Loss и Take Profit
- Максимальный дневной убыток (5%)
- Максимальная просадка от пика (15%)
- Автоматическое сокращение позиции после серии убытков
- Trailing Stop (фиксированный %)

**Бэктестинг**
- Тестирование стратегий на исторических данных Binance
- Сравнение нескольких стратегий на одних данных
- Визуализация: equity curve, drawdown, win rate, profit factor
- Excel-отчёты с детальной информацией по каждой сделке (xlsx)
- Поддержка произвольных дат и торговых пар

**Telegram-интерфейс**
- Полное управление ботом через Telegram
- Запуск/остановка, смена стратегии, уровня риска, торговых пар
- Бэктест и сравнение стратегий прямо в чате
- Справка по командам и подробное описание стратегий
- Excel-отчёты отправляются файлом в чат

**Инфраструктура**
- Docker и docker-compose для быстрого развёртывания
- SQLite для хранения сделок и истории баланса
- Конфигурация через .env (Pydantic validation)
- Логирование в файл и консоль

### Проведённые исследования

- [Анализ топ open-source ботов](research.md) (Freqtrade, Hummingbot, Jesse, OctoBot) -- стратегии, функции, лучшие практики
- [Alpha Arena: LLM-трейдинг](alpha-arena.md) -- как нейросети торгуют криптой в реальном времени, оценка затрат на API
- [Классика vs LLM](classic-vs-llm.md) -- сравнительный анализ двух подходов

## Фаза 1: Улучшение текущих стратегий (1-2 недели)

> Цель: повысить прибыльность и реалистичность существующих стратегий.

### 1.1. ATR-based Trailing Stop
- **Что:** замена фиксированного % trailing stop на адаптивный, основанный на ATR (Average True Range)
- **Зачем:** фиксированный % не учитывает текущую волатильность -- на спокойном рынке стоп слишком далеко, на волатильном -- выбивает раньше времени
- **Эффект:** меньше ложных срабатываний, лучшее соотношение risk/reward
- **Сложность:** низкая

### 1.2. Slippage в бэктесте
- **Что:** добавить моделирование проскальзывания (0.05-0.1%) при исполнении ордеров в бэктесте
- **Зачем:** текущие результаты бэктеста завышены на 20-40% относительно реальной торговли
- **Эффект:** результаты бэктеста ближе к реальности, более точные ожидания
- **Сложность:** низкая (буквально одна строка в расчёте PnL)

### 1.3. Geometric Grid
- **Что:** замена arithmetic grid (равный шаг в $) на geometric grid (равный шаг в %)
- **Зачем:** geometric grid лучше соответствует природе ценовых движений -- процентные изменения, а не абсолютные
- **Эффект:** более равномерное распределение прибыли по уровням
- **Сложность:** низкая

### 1.4. Multi-Timeframe подтверждение
- **Что:** стратегии анализируют не один таймфрейм, а два-три (например, сигнал на 1H + подтверждение на 4H)
- **Зачем:** сигналы, подтверждённые на старшем таймфрейме, имеют значительно более высокий win rate
- **Эффект:** меньше ложных сигналов, выше процент прибыльных сделок
- **Сложность:** средняя

## Фаза 2: LLM-стратегия (1-2 недели)

> Цель: добавить нейросеть как 7-ю стратегию и сравнить с классическими.

### 2.1. Стратегия `llm_trader`
- **Что:** новая стратегия, которая вместо индикаторов отправляет рыночные данные в API нейросети
- **Модели:** Claude (Anthropic), GPT (OpenAI), DeepSeek -- выбор через .env
- **Промпт:** структурированный системный промпт с правилами risk management, формат ответа -- JSON
- **Fallback:** если API недоступен -- пропускаем цикл, не торгуем

### 2.2. Промпт-инжиниринг
- **Что:** разработка и тестирование системного промпта для торговли
- **Ключевое:** жёсткие ограничения в промпте (макс. плечо, макс. размер позиции, обязательный SL/TP)
- **Урок Alpha Arena:** модели сливают не из-за плохих предсказаний, а из-за отсутствия правил risk management в промпте

### 2.3. Paper Trading сравнение
- **Что:** запуск LLM-стратегии параллельно с классическими на paper trading
- **Длительность:** 1-2 месяца
- **Результат:** объективное сравнение подходов на одних и тех же рыночных условиях

## Фаза 3: Оптимизация и автоматизация (2-4 недели)

> Цель: автоматический подбор параметров и защита от переобучения.

### 3.1. Hyperopt через Optuna
- **Что:** автоматическая оптимизация параметров стратегий (пороги RSI, периоды EMA, ATR-множители)
- **Зачем:** ручной подбор параметров неэффективен, оптимизация может превратить убыточную стратегию в прибыльную
- **Реализация:** интеграция библиотеки Optuna, loss-функции: Sharpe Ratio, Sortino, Calmar
- **Сложность:** средняя

### 3.2. Walk-Forward Testing
- **Что:** бэктест с разделением на train/test окна -- оптимизация на одном периоде, проверка на следующем
- **Зачем:** защита от переобучения (overfitting) -- стратегия, которая идеально работает на исторических данных, может провалиться в реальности
- **Сложность:** средняя

### 3.3. WebSocket вместо Polling
- **Что:** переход с периодического опроса API биржи на WebSocket-подписки
- **Зачем:** real-time данные без задержек, меньше нагрузка на API
- **Сложность:** средняя

## Фаза 4: Продвинутые стратегии (1-2 месяца)

> Цель: добавить стратегии, которые используют лидеры рынка.

### 4.1. Market Making
- **Что:** стратегия выставления bid/ask ордеров с заработком на спреде
- **Требования:** капитал от $5,000, понимание inventory risk
- **Референс:** Hummingbot -- лидер open-source market making

### 4.2. Арбитраж
- **Что:** обнаружение и эксплуатация разницы цен между биржами или парами
- **Виды:** кросс-биржевой, треугольный (внутри одной биржи)
- **Ограничение:** требует минимальной латентности, на Python сложно конкурировать

### 4.3. Sentiment Analysis
- **Что:** анализ настроений рынка через Google Trends, Reddit, Twitter
- **Роль:** не самостоятельная стратегия, а фильтр/усилитель для существующих
- **Инструменты:** FinBERT (open-source NLP для финтекста)

## Фаза 5: Продакшен (постоянно)

> Цель: переход от paper trading к реальной торговле.

### 5.1. Testnet торговля
- **Что:** торговля на testnet-аккаунтах Binance/Bybit с реальным исполнением ордеров (но виртуальными деньгами)
- **Зачем:** проверка что бот корректно работает с реальным API биржи (ордера, балансы, ошибки)

### 5.2. Live торговля с минимальным капиталом
- **Что:** запуск на реальном счёте с $50-100
- **Зачем:** проверка на реальных комиссиях, slippage, задержках
- **Правило:** только после успешного paper trading минимум 1 месяц

### 5.3. Мониторинг и алерты
- **Что:** уведомления в Telegram о критических событиях (ликвидация, drawdown > порога, ошибки API)
- **Зачем:** бот может работать автономно, но оператор должен знать о проблемах
